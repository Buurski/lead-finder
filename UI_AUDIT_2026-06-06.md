# UI Audit — Command Center (2026-06-06)

**Metode:** Hver side besøgt live på prod (`lead-finder-three-beta.vercel.app`)
i mobil-viewport (390×844, iPhone-størrelse) via Playwright, + statisk kode-scan
for ghost-features (døde knapper, nav→slettet side, placeholders). Desktop-layout
spot-tjekket via samme komponenter.

**Auth-note:** Første load med basic-auth sætter `cc_sess`-cookie; derefter loader
sider rent uden credentials i URL'en (vigtigt — se Non-findings #1).

**Dækning:** 16 ruter besøgt — `/welcome /` (Mission Control via welcome-gate),
`/leads /leadgen /approve /replies /messenger /clients /claude /spend /goals /seo
/studio /journal /memory /build-guide /settings`. Alle loader + autentificerer OK.

---

## Findings

### P1 — Reelle bugs

**1. `/build-guide` (Plan-historik): 7 af 8 dokumenter kunne ikke læses.**
Cleanup 1de07ec flyttede nat-rapporterne + planerne til `docs/archive/`, men
`DOCS`-arrayet pegede stadig på repo-roden → siden viste 1 ægte doc
(`COMMAND_CENTER_VISION.md`) + 7 røde "Kunne ikke læse X.md"-rækker. Reelt en
næsten-død side efter oprydningen.
→ **FIXET i denne session:** `readDoc()` prøver nu rod først, så `docs/archive/`
som fallback. Alle 8 docs vises igen.

### P2 — Mobil-polish (layout)

**2. "Spørg Claude"-FAB overlapper bund-indhold på hver side.**
Den flydende chat-knap nederst-til-højre dækker kort-actions og sidste liste-række:
på `/messenger` dækker den "Marker sendt"/"Spring over" + pagination; på `/leads`,
`/replies`, `/clients`, `/goals` dækker den sidste synlige række. Forslag: tilføj
bund-padding på `.cc-content` (fx `padding-bottom: 88px` på mobil) eller løft FAB'en.

**3. Horisontal overflow på data-tunge flader (mobil).**
- `/leads`: stats-kortet har et kort der "kigger" ud i højre kant; tabel-header
  (`RANG VIRKSOMHE…`) er beskåret.
- `/replies`: "Scan nu"-knappen er klippet af kortets højre kant.
- `/build-guide`: brede markdown-tabeller + `code`-spans ombrydes ikke → indhold
  beskæres ("Google Shee…", "no UI is built in this phas…").
Forslag: `overflow-x:auto` på tabel-wrappers + `word-break`/`overflow-wrap` på
`code`-spans i `MarkdownLite`; lad smalle action-knapper wrappe i stedet for at
flyde ud.

### P3 — Konsistens / mindre

**4. Inkonsistente sidetitler (`<title>`).**
`/leads`, `/clients`, `/approve` viser generisk "Command Center", mens
`/messenger /replies /leadgen /settings /spend /goals /seo /studio /journal
/memory /build-guide` har "X · Command Center". → Tilføj `export const metadata`
til de tre sider.

**5. `/memory` badge "Vault: live" men kilde er LOCAL.**
Memory-siden viser "Vault: live" men `KILDE: LOCAL` (build-tids-snapshot af den
bundtede vault) — mens `/journal` viser `KILDE: REMOTE` (live GitHub contents-API).
På prod opdaterer memory derfor IKKE live. → Enten wire memory til remote som
journal, eller omdøb badge til "lokal snapshot" så det ikke overlover.

**6. "ls"-mærke stadig på `/welcome`-hero.**
Sidebar-brandet blev ændret fra "ls" til accent-dot (57d8e76), men welcome-heroens
store mærke viser stadig "ls" (Lucas-initialer). Lucas droppede "Lucas" fra
branding → match welcome-mærket.

**7. `/klienter` viser rå lokal Windows-sti.**
Klient-kort viser `C:\Users\Buur\Clients\Vestfjends VVS` (Drive/mappe-stien). Lidt
upoleret i et web-UI. → Vis kun mappenavn, eller skjul stien bag et ikon/hover.

## Non-findings (undersøgt, ikke en bug)

1. **Console-fejl "Request cannot be constructed from a URL that includes
   credentials" (`/leads`, `/messenger`).** Artefakt af MIN audit-metode: jeg
   loadede med `user:pass@`-URL, hvilket forgifter `window.location` så relative
   `fetch()` fejler. Rigtige brugere logger ind via browserens basic-auth-dialog
   (Authorization-header, ingen creds i URL) → sker ikke. Verificeret: gen-load
   uden creds → `/api/messenger` hentede fint.
2. **`/hermes` "snart" + Hermes "kommer snart"-kort.** Ærlig placeholder
   (`soon:true` i nav-config), ikke en ghost — siger selv at den ikke er bygget.
3. **`next.config.ts` Turbopack NFT-warning ved build.** Eksisterende
   build-warning (whole-project-trace), ikke UI, ikke fra denne audit.
