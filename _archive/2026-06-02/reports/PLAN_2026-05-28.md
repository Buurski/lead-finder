## A. Granulær pause-kontrol

**Mål:** erstat ét binært halt-flag med tre uafhængige toggles +
master kill-switch. Hver send-path tjekker SIN egen flag.

### Datalag (Sheets)

Behold `PauseSchedule!A2` som **master kill** (bagudkompatibelt).
Tilføj nye celler:

```
PauseSchedule!A1  PausedUntil        (existing — master kill)
PauseSchedule!B1  SetAt              (existing — master timestamp)
PauseSchedule!C1  PausedCold         (new — ISO timestamp eller tom)
PauseSchedule!D1  PausedFollowup     (new)
PauseSchedule!E1  PausedManual       (new)
PauseSchedule!F1  SetByCold          (new — timestamp metadata)
PauseSchedule!G1  SetByFollowup      (new)
PauseSchedule!H1  SetByManual        (new)
```

Pause-logik per type:
- `paused if master.until > now OR specific.until > now`
- Tom celle / parseFejl på SPECIFIK type ≠ paused (kun master fejler CLOSED)
- Sentinels (`indefinite|forever|stop|halt`) virker på alle 4

### Code changes

| File | Change |

| `src/lib/eligibility.ts` | No change — eligibility is about lead-state, not send-state. The pause check stays in routes. |
| `src/app/api/email/bulk-send/route.ts` | `getPauseStatus("all")` AND `getPauseStatus("cold")` — bail if either is paused. Mid-run recheck same. |

## B. Spacing-garanti i sending-pipelinen

**Mål:** ingen Vercel-route må sende uden minimum 4-min-jitter mellem mails. Lokal `send.mjs` skal forblive single source of truth for spacing.

### Strategi: route alt via send.mjs's filsystem-kø

Vercel-routes må IKKE kalde Gmail direkte. I stedet:

1. **Enqueue til Sheets-tab `SendQueue`** (filsystem virker ikke på Vercel — bruger Sheets som message broker).
2. Lokal `send.mjs` poller `SendQueue`-tabben hver 60s og APPENDER til `queue.json`.
3. send.mjs sender med dens eksisterende 4–14 min spacing.

### Sheets-tab `SendQueue` schema

```
A: id (uuid)
B: enqueuedAt (ISO)
C: leadId
D: kind ("cold" | "followup")
E: subject
F: body  (full text — not template ID, så Lucas's review-godkendelse er
         låst til den eksakte body han så)
G: htmlBody  (for tracking pixel)
H: status ("pending" | "claimed" | "sent" | "skipped" | "expired")
I: claimedAt
J: sentAt
K: skipReason (hvis skipped efter enqueue)
```

## D. Auto-refresh på /review

**Mål:** liste opdateres automatisk når Lucas skipper/godkender på telefon mens en anden tab er åben, og når nye items enqueues.

### Strategi: simpel polling med React Query OR plain useEffect+setInterval

Næsten gratis at lave med fetch + setInterval(30000). React Query
er overkill her. Brug client-side state der re-fetcher
`/api/review/queue?json=1` (ny endpoint som returnerer serialised
queue.entries — samme format som server-component sender ind).

### Code changes

| File | Change |
|---|---|
| `src/app/api/review/queue/route.ts` (ny) | GET returnerer `{entries, summary, overflow, paused, pausedUntil, pauseStates: {cold, followup, manual}}` — alt der bruges af UI. `force-dynamic`, no cache. |
